I embarked on a 30-day challenge to master the Linux ecosystem. My approach was broken into 3 distinct phases to ensure I didn't just memorize commands, but understood the "why" behind them.

Learning Roadmap
1. Foundation
Immersive YouTube tutorials and documentation
The focus was to understand the Linux Kernel, Filesystem Hierachy and the philosophy of "Everything is a file". Learning how bash interprets commands and how the OS manages hardware via drives.

2. Implementation (Hands-on)
I used interactive labs via LabEx. The main focus was to do practical application in a sandboxed environment.
Key skills acquired are: User & Group Management (chmod,chown, sudoers), Process Monitoring (top, ps), Network Configuration (ifconfig, ip a, netstat), Package Management and Repository handling.

3. Mastery (Exploitation & Troubleshooting)
I used "Exploiting labs." Focus was on security-focused challenges and system hardening.
Goals achieved were finding vulnerabilities, understanding misconfigurations, and learning how to "break" the system to better understand how to protect it.

I didn't just learn how to use Linux; I learned how to think in Linux
